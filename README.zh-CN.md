## 为什么 2026 年 Polymarket 跟单很重要

Polymarket 的公开订单流与公开钱包在 **每个事件品类** 形成了「聪明钱」层：

- 顶级交易者通过 **链上成交** 公开优势，而非私密信号
- 手动跟随太慢 — 你点击时盘口已在移动
- 跟单机器人 **7×24** 自动检测 → 过滤 → 定仓 → 执行
- 真正竞争是 **能否跟得好**，而非「能不能跟」

行业模式（Telegram 跟单、托管平台、开源引擎）：

- 盲跟复制 **噪音**（灰尘单、对冲、库存抛售）
- 评分 / 选择性跟单通过跳过低信号成交提升预期质量
- 比例定仓在不同资金规模下对齐风险
- 延迟与滑点决定你能否拿到领袖价格 — 还是更差价格
- 出场策略（镜像 vs 自有 TP/SL）往往与入场同样重要

因此现代 Polymarket 跟单系统是 **策略栈**，而非单一的「跟随这条鲸鱼」开关。

## 核心策略手册

### 1. 智能 / 评分钱包跟单

领袖进入跟单池前评分：

| 标准 | 衡量内容 | 为何重要 |
|------|----------|----------|
| **胜率** | 已结算盈利仓位占比 | 过滤抛硬币式赌徒 |
| **平均 ROI** | 每已结算市场平均回报 | 过滤高量低边际交易者 |
| **市场多样性** | 交易品类 / 唯一市场数 | 避免单一赛道运气 |
| **活跃度** | 距上次交易天数 | 剔除失效钱包 |
| **校准度** | 价格是否长期匹配结果 | 区分技能与运气 |

动态池：加入合格者；剔除衰减者；领袖被剔除时在 Telegram/Discord 提醒。

### 2. 选择性信号跟单

即使优秀钱包也会做 mediocre 交易。对每笔成交评分（市场类型、相对规模、时机、盘口质量），只复制高信念信号。

### 3. 全量镜像跟单

选定钱包并镜像每次买卖。简单基线 — 通常还需定仓上限与灰尘过滤。

### 4–6. 定仓：比例 / 固定 / 自适应

| 模式 | 思路 | 示例 |
|------|------|------|
| **比例** | 复制风险份额 | 领袖买 $1,000 → 你买 10% = $100 |
| **固定** | 每信号相同 USD | 每笔合格信号 $50 |
| **自适应** | 计算后钳制 | 算出 $320 → 上限 $100 → 执行 $100 |

硬保护：每笔/市场/小时/日上限、最大持仓数、日亏损 / 回撤熔断。

### 7–8. 多钱包与共识

跟随 **3–7 名专家**，设 per-wallet 上限。可选共识：仅当多名领袖在时间窗内同向时入场。

### 9. 出场镜像 vs 独立 TP/SL

| 模式 | 行为 |
|------|------|
| **镜像出场** | 领袖卖出/减仓时你也跟随 |
| **独立 TP/SL** | 自有止盈 / 止损 / 时间止损 |
| **混合** | 默认镜像 + 硬止损 / 日熔断 + 可选止盈 |

特色方案：**定仓 + 可选 TPSL**，而非纯盲目继承出场。

### 10–12. 过滤：品类、延迟、时间窗

- **品类过滤** — 仅政治 / 体育 / 加密等领袖有边际的赛道
- **延迟优先 / 内存池** — 链上尽早检测，避免以更差价格跟同一思路
- **时间窗** — 短市场（5m/15m）的早盘狙击或尾盘窗口

### 13–15. 生命周期与执行质量

- **仅买入** vs **买卖全镜像**（`COPY_SELLS` 式控制）
- **限价偏移 / 最大滑点 / FAK** — 不无限追价
- **灰尘 / 最小成交过滤** — 忽略领袖低于 e.g. $10–$50 的成交
